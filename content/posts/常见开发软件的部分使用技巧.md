## Git

### 让 `git status` 显示中文文件名

默认情况下，Git 会对非 ASCII 文件名（如中文）做八进制转义显示。在 PowerShell 或 Git Bash 里跑 `git status`，中文目录和文件名会变成一串难读的转义码：

```text
modified:   "\344\270\252\344\272\272\347\254\224\350\256\260/..."
```

**原因**：Git 的 `core.quotepath` 选项默认开启，对非 ASCII 路径进行 quote 转义，导致中文显示为八进制序列。

**解决**：关闭该选项，Git 即可直接输出原始 UTF-8 中文：

```bash
# 仅对当前仓库生效
git config core.quotepath false

# 或全局生效（推荐，一次解决所有仓库）
git config --global core.quotepath false
```

设置后，同样的 `git status` 输出变为：

```text
modified:   个人笔记/个人理财/经济学入门.md
```

中文文件名正常显示。

### 用 Syncthing 同步 Git 仓库时的冲突副本

如果像我一样用 Syncthing 等同步工具在多设备间同步 Git 仓库，仓库根目录可能冒出 `.sync-conflict-2026xxxx-xxxxxx-XXXXXX.gitignore` 这样的冲突副本。这是同步工具在两端都检测到改动、无法自动合并时生成的保留文件。处理方式是先用 `diff` 比对两份内容，确认当前 `.gitignore` 已是完整最新版本后，直接删除冲突文件即可，无需手动合并。

### Gerrit 通过 SSH 克隆仓库报 Permission denied (publickey)

使用 Gerrit 这类代码评审平台时，常会直接复制这样的克隆命令：

```bash
git clone ssh://<gerrit-host>:29418/<repo-name>
```

如果照搬，很容易遇到如下报错：

```text
<当前系统登录名>@<gerrit-host>: Permission denied (publickey).
fatal: 无法读取远程仓库。

请确认您有正确的访问权限并且仓库存在。
```

**原因**：SSH 地址里没有指定用户名时，SSH 客户端会默认使用**当前操作系统的登录用户名**去尝试认证。但 Gerrit 的 SSH 账号名是你在 Gerrit Web 上注册的账号（通常是邮箱前缀或工号），与 Linux 主机的登录名往往不是一回事。用错用户名去撞认证，自然每次都被拒绝，报 `publickey` 错误。

**解决**：在 SSH 地址中显式带上 Gerrit 账号名即可：

```bash
git clone ssh://<your-gerrit-username>@<gerrit-host>:29418/<repo-name>
```

指定正确用户名后，认证就能通过，克隆正常进行。

**补充排查**：如果带上正确用户名后仍报 `publickey` 错误，则问题大概率在本机公钥未登记到 Gerrit。按以下顺序检查：

1. 本地生成密钥（若尚无）：`ssh-keygen -t rsa -b 4096 -C "your-email@example.com"`
2. 取出公钥内容：`cat ~/.ssh/id_rsa.pub`，复制整行
3. 登录 Gerrit Web → **Settings → SSH Keys**，粘贴并添加
4. 验证连通性：

```bash
ssh -p 29418 <your-gerrit-username>@<gerrit-host> gerrit version
```

返回 `gerrit version X.Y.Z` 即表示认证成功，之后 clone / push 都不会再被拦。

> 提示：Gerrit 默认 SSH 端口是 `29418`（区别于标准 22 端口），地址里务必带对端口号。

## PowerShell / 终端

### 解决 PowerShell 红字蓝底看不清

在经典 PowerShell（非 Windows Terminal）中，`git status` 的 modified 文件默认用红色显示，而 PowerShell 的默认背景是深蓝色（`#012456`）。红字配蓝底，对比度极差，几乎没法读。

有以下四种解法，按推荐程度排序：

**方案一（推荐）：改用 Windows Terminal**

Windows 10/11 自带，按 `Win + R` 输入 `wt` 回车即可打开。默认黑色背景，红色文字清晰可辨，还支持多标签和自定义主题。把快捷方式固定到任务栏，以后默认用它替代 PowerShell 即可。

**方案二（永久）：修改 PowerShell 默认配色**

右键点击 PowerShell 窗口标题栏 → 属性 → 颜色 → 选中「屏幕背景」→ 改为黑色 → 确定时选择「修改启动该窗口的快捷方式」。这样以后每次打开都会生效。

**方案三（最快）：临时改当前会话背景色**

```powershell
$Host.UI.RawUI.BackgroundColor = "Black"
Clear-Host
```

输入后背景立即变黑，但只影响当前窗口，关闭即失效。

**方案四（Git 层面）：修改 Git 输出颜色**

让 Git 把 modified 文件从红色改为黄色，在蓝底上也能看清：

```bash
git config --global color.status.changed "yellow normal"
git config --global color.status.untracked "cyan normal"
```

此方案治标不治本——其他命令行工具的红字蓝底问题依旧存在。

**小结**：Windows Terminal 或把 PowerShell 背景改成黑色是从根本上解决问题；改 Git 颜色只是绕开显示问题。
