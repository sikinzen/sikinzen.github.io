## 背景

在"[零端口暴露，5 分钟打通内网：Tailscale 内网穿透实战](/posts/tailscaleintranetpenetration/)"和"[蒲公英 vs Tailscale：贝锐蒲公英内网穿透实战与方案对比](/posts/oraypgyintranetpenetration/)"两篇文章中，我已经把内网穿透跑通了——家里和公司电脑之间建立了一条低延迟的加密隧道。

隧道通了之后，接下来要解决的实际问题是：**从家里访问公司电脑上的文件夹，用什么方式最好？**

这篇文章实测对比四种主流方案，给出结论。

---

## 四种方案速览

| 方案 | 一句话概括 | 需要穿透吗 |
|------|-----------|-----------|
| **向日葵远程文件传输** | 远程桌面软件自带的文件管理器，拖拽就能传 | ❌ 自带穿透 |
| **Windows SMB 共享** | 把公司文件夹映射为本地的 Z: 盘 | ✅ 依赖蒲公英/Tailscale |
| **Syncthing** | P2P 自动同步，两台机器互为镜像 | ✅ 依赖蒲公英/Tailscale |
| **WebDAV** | HTTP 协议扩展，轻量级文件访问 | ✅ 依赖蒲公英/Tailscale |

## 方案一：向日葵远程文件传输

### 原理

向日葵本质是一个**远程桌面工具**，但内置了文件传输模块。连接远程电脑后，打开文件传输窗口就能双向操作两台机器的文件系统。

```
笔记本                        公司电脑
 ┌──────────┐                ┌──────────┐
 │ 向日葵客户端 │  ←── 加密隧道 ──→ │ 向日葵客户端 │
 │            │                │  (被控端)   │
 │ 文件传输窗口 │                │            │
 │ 拖拽/复制  │                │            │
 └──────────┘                └──────────┘
```

不需要任何额外的穿透工具——向日葵自带内网穿透，和蒲公英、Tailscale 是平行关系，而不是附属关系。

### 核心功能

| 功能 | 说明 |
|------|------|
| **双向传输** | 笔记本 → 公司、公司 → 笔记本，拖拽文件即可 |
| **远程文件管理器** | 直接浏览公司电脑的目录结构，像本地资源管理器一样 |
| **断点续传** | 文件中断后自动继续，不用重新传 |
| **文件夹传输** | 支持整文件夹拖拽 |
| **共享文件夹** | 设置指定文件夹为共享态，多端可访问 |
| **传输速度** | 走向日葵自有协议，国内服务器加速，默认高速模式 |

### 优点

1. **自带穿透**：不需要蒲公英/Tailscale，一个向日葵搞定远程桌面 + 文件传输
2. **上手最简单**：拖拽文件，零配置，不需要设共享、不需要映射磁盘
3. **功能最全**：除了文件传输，还能远程桌面操作公司电脑（一站式解决）
4. **速度快**：国内服务器优化，共享文件夹模式下支持多端同步
5. **权限控制**：可设置只读/可写，适合团队场景

### 缺点

1. **公司电脑必须开机且登录**：向日葵是远程桌面，被控电脑锁屏或被注销就无法操作
2. **免费版有限制**：个人免费版带宽和传输速度受限，高速传输需付费
3. **需要安装客户端**：两端都要装向日葵
4. **每次都要"远程"进去**：不是常驻映射，每次操作都要先打开向日葵，不是"无缝"体验

## 方案二：Windows SMB 共享

### 原理

SMB（Server Message Block）是 Windows 原生的文件共享协议。把公司电脑上的文件夹设为共享，再通过蒲公英/Tailscale 的虚拟网络隧道，在笔记本上映射为网络驱动器（如 Z: 盘）。

```
笔记本（Z: 盘）               蒲公英/Tailscale 隧道            公司电脑
 ┌─────────────────┐          ┌──────────────┐          ┌─────────────────┐
 │ \\10.168.x.x\   │  ←──→   │ 加密隧道       │  ←──→   │ D:\CompanyFiles │
 │   CompanyFiles  │          │ 延迟 20-50ms  │          │ (共享文件夹)     │
 └─────────────────┘          └──────────────┘          └─────────────────┘
```

### 搭建步骤

在公司电脑上：

```powershell
# 1. 创建共享入口文件夹
mkdir D:\CompanyFiles

# 2. 右键 → 属性 → 共享 → 添加你的用户 → 勾选"读取/写入"
# （或用命令行）
net share CompanyFiles=D:\CompanyFiles /grant:你的用户名,FULL

# 3. 把散落各处的目录用符号链接汇聚
mklink /D "D:\CompanyFiles\项目文档" "D:\Projects\Docs"
mklink /D "D:\CompanyFiles\技术资料" "E:\TechDocs"

# 4. 放行蒲公英/Tailscale 网段的 SMB（445 端口）
netsh advfirewall firewall add rule name="Allow SMB from VPN" dir=in action=allow protocol=TCP localport=445 remoteip=10.168.0.0/16
```

在笔记本上：

```powershell
# 映射为 Z: 盘（用公司电脑的蒲公英/Tailscale 虚拟 IP）
net use Z: \\10.168.x.x\CompanyFiles /persistent:yes
```

### 优点

1. **无缝体验**：映射后 Z: 盘就是本地盘——双击打开文件、Ctrl+S 保存，和在公司没区别
2. **零额外软件**：Windows 原生，不需要装任何东西
3. **不占硬盘**：按需访问，不消耗笔记本存储空间
4. **权限精细**：可设只读/读写/仅特定用户

### 缺点

1. **依赖穿透**：必须先有蒲公英或 Tailscale 打通隧道，SMB 本身不管穿透
2. **高延迟下体验差**：如果隧道速度慢（如 Tailscale relay 模式），小文件操作会有明显延迟
3. **大文件慢**：在网络盘上直接编辑大文件不如先拉到本地
4. **需要初期配置**：设共享、开防火墙、建符号链接、映射驱动器——首次部署 10-15 分钟

## 方案三：Syncthing P2P 同步

### 原理

Syncthing 在两台机器之间建立 P2P 连接，把你选中的文件夹自动同步到对方机器上。不是"远程访问"，而是"你有，我也有"。

```
笔记本                        蒲公英/Tailscale 隧道            公司电脑
 ┌─────────────────┐          ┌──────────────┐          ┌─────────────────┐
 │ D:\CompanyFiles │  ←──→   │ 加密隧道       │  ←──→   │ D:\CompanyFiles │
 │ （本地副本）      │          │              │          │ （主副本）       │
 │                 │          │  自动同步      │          │                 │
 └─────────────────┘          └──────────────┘          └─────────────────┘
```

### 核心特性

| 特性 | 说明 |
|------|------|
| **双向同步** | 公司改了，笔记本自动更新；笔记本改了，公司也更新 |
| **版本保留** | 可配置保留 N 个历史版本，防误删 |
| **增量传输** | 只传改动部分，不是全量拷贝 |
| **选择性同步** | 可以只同步某个子目录，不必全量 |

### 优点

1. **本地读写速度**：文件就在笔记本硬盘上，打开、编辑都秒级响应
2. **离线可用**：同步完成后断网也能使用，联网后自动合并
3. **备份属性**：本质上也是备了一份在公司电脑上
4. **开源免费**：永久免费，无任何限制

### 缺点

1. **占用本地硬盘**：同步多少就占多少空间
2. **初次同步慢**：文件量大时，首次全量同步需要很长时间
3. **冲突处理**：两人同时改同一文件会产生冲突副本，需要手动合并
4. **也依赖穿透**：需要蒲公英/Tailscale 提供网络通路

## 方案四：WebDAV

### 原理

WebDAV 是基于 HTTP 协议扩展的文件管理协议。在公司电脑上搭建 WebDAV 服务端，笔记本通过 RaiDrive 等工具映射为网络驱动器。

### 优点

1. 比 SMB 更轻量，穿透性好
2. 支持 HTTPS 加密
3. 可通过浏览器直接访问

### 缺点

1. Windows 自带 WebDAV 客户端体验差（需用 RaiDrive 等第三方工具）
2. 需额外搭建 WebDAV 服务端
3. 目录浏览和搜索不如 SMB
4. 综合体验不如前三种方案

## 七维度完整对比

| 维度 | 向日葵 | SMB | Syncthing | WebDAV |
|------|:------:|:---:|:---------:|:------:|
| **部署难度** | ⭐⭐⭐⭐⭐ 装客户端即用 | ⭐⭐⭐ 需配共享+防火墙 | ⭐⭐⭐ 装客户端+配同步 | ⭐⭐ 需搭服务端 |
| **使用体验** | ⭐⭐⭐⭐ 拖拽直观 | ⭐⭐⭐⭐⭐ 像本地盘 | ⭐⭐⭐⭐⭐ 纯本地速度 | ⭐⭐⭐ 勉强可用 |
| **传输速度** | ⭐⭐⭐⭐⭐ 自有加速 | ⭐⭐⭐⭐ 取决于隧道 | ⭐⭐⭐⭐ P2P 同步 | ⭐⭐⭐ 取决于隧道 |
| **不占本地硬盘** | ✅ | ✅ | ❌ 占硬盘 | ✅ |
| **自带宽敞穿透** | ✅ | ❌ 依赖蒲公英/Ts | ❌ 依赖蒲公英/Ts | ❌ 依赖蒲公英/Ts |
| **远程桌面一体** | ✅ 一站式 | ❌ | ❌ | ❌ |
| **离线可用** | ❌ | ❌ | ✅ | ❌ |
| **免费程度** | ⭐⭐⭐ 限速 | ⭐⭐⭐⭐⭐ 完全免费 | ⭐⭐⭐⭐⭐ 完全免费 | ⭐⭐⭐⭐ 需工具 |
| **权限控制** | ⭐⭐⭐⭐ 精细 | ⭐⭐⭐⭐ 精细 | ⭐⭐ 简单 | ⭐⭐⭐ 一般 |
| **适合大文件** | ⭐⭐⭐⭐⭐ 断点续传 | ⭐⭐⭐ 一般 | ⭐⭐⭐ 初同步慢 | ⭐⭐ 差 |
| **适合小文件多** | ⭐⭐⭐ 逐个传输 | ⭐⭐⭐ 每个建连 | ⭐⭐⭐⭐ 增量同步 | ⭐⭐⭐ |

## 选型建议

```
场景                              推荐方案
────────────────────────────────────────────────────
主要用远程桌面，顺便传文件             → 向日葵（一站式）
看文档/Excel，偶尔编辑保存            → SMB（最像本地）
每天大量读写，离线也要用               → Syncthing
传输大文件/固件包                    → 向日葵（断点续传+加速）
团队共享文件夹，权限管控               → 向日葵（企业版）或 SMB
零成本，已有穿透隧道                  → SMB（原生免费）
```

## 我的选择与使用方式

**主力方案：向日葵远程文件传输**

理由很直接：

1. 我已经有向日葵了（用来远程桌面操作公司电脑），文件传输是顺带功能
2. 拖拽传文件比我之前用 Tailscale relay 模式下的 SMB 快得多
3. 断点续传对大文件友好（固件包、编译产物几百 MB 到几 GB）
4. 不需要额外配共享文件夹、放防火墙、映射驱动器——直接用

**备用方案：SMB**（通过蒲公英硬件网关隧道）

等 SMB 搭好后作为补充：当不需要远程桌面、只想快速打开几个文档时，Z: 盘就是最方便的。两者不是互斥，而是互补。

## 总结

| 问题 | 结论 |
|------|------|
| 哪种最快？ | 向日葵（自有加速）+ Syncthing（本地读写） |
| 哪种最方便？ | 向日葵（拖拽即传）+ SMB（映射为本地盘） |
| 哪种最省钱？ | SMB / Syncthing（完全免费） |
| 哪种功能最全？ | 向日葵（远程桌面 + 文件传输 + 权限管理一体） |
| 我选哪个？ | 向日葵主力 + SMB 备用 |

内网穿透解决"怎么连"，文件访问解决"怎么用"。两条路都通了，才能真正实现"在家和在公司没区别"。
